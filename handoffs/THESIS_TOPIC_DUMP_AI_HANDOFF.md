## 3. Thesis Requirements From the Uploaded Guidelines

The user previously uploaded Bicol University / CS thesis guideline files. The important requirements extracted from those files are:

### A. A CS thesis must have a real computing contribution
A thesis should not be just a generic information system, CRUD app, portal, or database. The software is only a vehicle. The thesis must contribute through computational methods such as:

- algorithms,
- AI/ML/DL models,
- NLP,
- computer vision,
- cybersecurity mechanisms,
- data analytics,
- recommender systems,
- optimization,
- simulation,
- HCI evaluation,
- IoT/edge intelligence,
- distributed/cloud/edge computing,
- blockchain/security mechanisms,
- knowledge representation/reasoning.

### B. Avoid plain system titles
Not acceptable unless they include a substantial computing component:

- student information system,
- enrollment system,
- payroll,
- inventory,
- POS,
- library system,
- reservation system,
- e-commerce,
- general mobile app,
- general database system,
- ordinary portal.

Example difference:

- Weak: `Medical Records System`
- Better: `Intelligent Medical Records System Using NLP-Based Clinical Note Classification and Retrieval`

### C. Recommended title format
Use this pattern when possible:

> **[Computing Technique/Approach] + [Problem Being Solved] + [Application Domain]**

Examples:

- `Explainable Machine Learning for Early Detection of Student Dropout Risk in Higher Education`
- `YOLO-Based Crop Disease Detection for Smallholder Farming Decision Support`
- `Retrieval-Augmented Generation for Academic Policy Question Answering in University Services`

### D. Every topic bank should include these fields
Whenever creating an Excel topic bank, include these columns or equivalent:

1. Rank
2. Category / Subfield
3. Proposed Title
4. Simple Taglish Explanation
5. CS Domain / Field
6. Computing Contribution
7. Algorithms / Techniques / Frameworks
8. BU Agenda
9. SDG
10. Beneficiary
11. Problem to be Addressed
12. Description / Background
13. Research Gap / Novelty
14. Expected Innovation
15. Proposed Solution
16. Expected Prototype Output
17. Possible Data Source / Respondents
18. Data Access Status
19. RRL Availability
20. Evaluation Metrics
21. System Evaluation Method
22. Possible Challenges / Limitations
23. Feasibility
24. Difficulty
25. Risk Level
26. Approval Chance
27. Group Recommendation
28. Priority Score
29. Equipment / Permission Needed
30. Ethical / Safety Notes
31. Sources / Dataset Notes
32. Next Action

### E. Evaluation expectations
Topic ideas should mention measurable evaluation. Examples:

For AI/ML/NLP/CV:

- Accuracy
- Precision
- Recall
- F1-score
- Macro-F1
- ROC-AUC
- MAE / RMSE / MAPE for forecasting
- mAP for object detection
- IoU / Dice for segmentation
- BLEU / ROUGE / BERTScore for summarization or generation, when appropriate
- Recall@k / MRR / nDCG for retrieval and recommender systems

For system/prototype evaluation:

- ISO/IEC 25010
- SUS
- user acceptance testing
- expert evaluation
- task completion time
- usability rating
